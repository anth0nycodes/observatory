```typescript
export const instrumentStep: Step = {
  name: "instrument",

  async shouldRun(ctx: WizardContext): Promise<boolean> {
    return !!ctx.framework && !ctx.completedSteps.includes("instrument");
  },

  async run(ctx: WizardContext): Promise<StepResult> {
    // Full implementation below
  },
};
```

**The run() method orchestrates this flow:**

**Step A: Collect codebase context**
- Call `collectCodebaseContext(ctx)` from `./utils/codebase-context.js`
- Show spinner: "Analyzing your codebase..."

**Step B: Try AI instrumentation (if authenticated)**
- Per D-01, AI is primary for ALL frameworks
- Skip AI if `ctx.keyProvided` (--key mode, no auth token) or `!ctx.accessToken`
- If AI eligible:
  - Show spinner: "Generating project-specific instrumentation..."
  - Fetch POST to `https://api.thecontext.company/api/cli/instrument` (or dev URL if apiKey starts with `dev_`)
    - Headers: `{ "Content-Type": "application/json", "Authorization": "Bearer ${ctx.accessToken}" }`
    - Body: `{ framework: ctx.framework, context: codebaseContext }`
  - Use Promise.race with 15-second timeout (per research anti-pattern: don't let user wait)
  - On success: parse response as `{ patches, metadata }`
  - On failure/timeout: log warning with `p.log.warn("AI instrumentation unavailable, using template...")`, fall through to template

**Step C: Fall back to templates (per D-03)**
- If AI returned no patches (failed, timed out, or skipped):
  - Call `getTemplate(ctx.framework!, ctx)` from `./utils/templates/index.js`
  - Convert TemplateResult.files to the same patch format as AI response

**Step D: Detect gotcha fixes (per D-08)**
- Call `detectGotchaFixes(ctx)` from `./utils/gotcha-fixes.js`
- Append these as additional FileChange items

**Step E: Build FileChange array and show diffs (per D-04)**
- Convert patches + gotcha fixes into FileChange[] format
- For "create" actions: oldContent = "" (new file)
- For "modify" actions: read the current file content as oldContent, use patch content as newContent
- Call `reviewAndApplyChanges(changes)` from `./utils/diff-display.js`
- This shows colored diffs and collects accept/reject per file

**Step F: Apply accepted changes**
- For each applied file:
  - Ensure parent directory exists (fs.mkdirSync with recursive: true)
  - Write the new content (fs.writeFileSync)
- Log summary: `p.log.success("Applied N file(s), skipped M file(s)")`
- If AI was used and metadata available, log: "Metadata wired: sessionId: {yes/TODO}, userId: {yes/TODO}, conversational: yes"

**Step G: Return result**
- Return `{ status: "completed", message: "Instrumented with {AI/templates}" }` if any files applied
- Return `{ status: "completed", message: "All changes skipped by user" }` if user rejected all

**Important details:**
- Base URL logic: If `ctx.apiKey?.startsWith("dev_")`, use `https://dev.thecontext.company` instead of `https://api.thecontext.company`
- The fetch timeout should use AbortController with 15s signal, not Promise.race (cleaner cleanup)
- Import `fs` from `"node:fs"` and `path` from `"node:path"`
- Use `* as p from "@clack/prompts"` and `pc from "picocolors"` for UI
