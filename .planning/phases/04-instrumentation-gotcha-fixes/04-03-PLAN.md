2. Create `packages/ts/liftoff/src/utils/diff-display.ts` with:

**Exports:**
- `FileChange` interface: `{ filePath: string; oldContent: string; newContent: string; description: string }`
- `formatColoredDiff(patch: string): string` — colors diff lines: green for `+`, red for `-`, cyan for `@@`, bold for `+++`/`---`, dim for context
- `reviewAndApplyChanges(changes: FileChange[]): Promise<{ applied: string[]; skipped: string[] }>` — for each change, generates unified diff via `createTwoFilesPatch`, shows it with `p.note(coloredDiff, label)`, prompts `p.confirm()` per file, returns lists of applied/skipped paths

**Implementation details per D-04:**
- Use `createTwoFilesPatch` from `"diff"` to generate unified diffs
- Use `picocolors` (imported as `pc`) for coloring — NOT chalk
- Use `@clack/prompts` (imported as `* as p`) for confirm prompts and note display
- For new files (oldContent === ""), show `CREATE` label in green
- For modified files, show `MODIFY` label in yellow
- Label format: `${pc.green("CREATE")} ${filePath}` or `${pc.yellow("MODIFY")} ${filePath}` followed by ` -- ${description}`
- If user cancels (p.isCancel), treat as skip
- Default confirm value: true (bias toward applying)

Per D-11 (conflict handling): If a file to be modified doesn't match expected oldContent (i.e., the file changed since context was collected), show a warning `"File has changed since analysis — review carefully"` but still show the diff and let user decide.

**Exports:**
- `CodebaseContext` interface matching the request contract:
  ```typescript
  interface ContextFile {
    path: string;       // relative to installDir
    content: string;    // truncated to MAX_FILE_LINES
    role: string;       // "entry-point" | "config" | "auth-pattern" | "framework-config" | "instrumentation"
  }
  
  interface CodebaseContext {
    framework: Framework;
    language: ProjectLanguage;
    typescript: boolean;
    srcDir: boolean;
    files: ContextFile[];
    existingInstrumentation: boolean;
    packageJson?: { dependencies: Record<string, string>; devDependencies?: Record<string, string> };
  }
  ```
- `collectCodebaseContext(ctx: WizardContext): CodebaseContext` — the main function

**Implementation:**
- Constants: `MAX_FILE_LINES = 100`, `MAX_TOTAL_CHARS = 8000`
- Helper: `addFile(relPath, role)` — reads file if exists, truncates to MAX_FILE_LINES, tracks totalChars, skips if over budget
- Always collect: `package.json` (deps only — strip scripts/other fields to save space), or `pyproject.toml`/`requirements.txt` for Python
- Framework-specific collection via switch on ctx.framework:
  - **nextjs-aisdk**: next.config.{js,mjs,ts} as "config", src/instrumentation.ts or instrumentation.ts as "instrumentation", find files in src/app/api/ containing `generateText` or `streamText` as "framework-config"
  - **claude-agent-sdk**: find files importing `@anthropic-ai/claude-agent-sdk` as "entry-point"
  - **langchain-ts**: find files importing `@langchain/core` or `langchain` as "entry-point"
  - **mastra**: find file containing `new Mastra(` as "framework-config"
  - **pi-mono**: find files importing `@anthropic-ai/pi-mono` as "entry-point"
  - **openclaw**: find files importing `openclaw` as "entry-point"
  - **custom-ts**: main entry from package.json "main" field as "entry-point"
  - **Python frameworks**: main.py, app.py, src/main.py as "entry-point"; pyproject.toml as "config"
- Metadata detection: Search for files containing "session", "userId", "auth" patterns — add up to 2 as "auth-pattern" role (per D-05, these help AI wire conservative metadata)
